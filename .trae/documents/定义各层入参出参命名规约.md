## 目标

* 为单体单模块 4 层架构（interfaces/application/domain/infrastructure）定义统一的“入参/出参”对象命名方式，避免 DTO 渗透与边界混乱。

## 命名规约（将写入规则文档）

### Interfaces（入站适配层）

* Web Controller：`XxxRequest` / `XxxResponse`（或 `XxxView`），只承载 I/O 与校验注解

* Facade/RPC（对其他模块）：`XxxRequest` / `XxxDTO`（与模块化单体规则保持一致）

* Listener（MQ 入站）：`XxxEvent` / `XxxConsumeResult`（如果需要显式结果），事件对象不复用 Web/RPC DTO

### Application（用例层）

* 命令：`XxxCommand`

* 查询：`XxxQuery`

* 用例输出：`XxxResult`（或 `XxxOutput`，二选一全局统一）

* 应用内部数据结构：`XxxDTO`（仅限 application 内部/对 interfaces 输出；不进入 domain）

### Domain（领域层）

* 领域模型：`Xxx`（Entity/AggregateRoot）、`XxxId`、`XxxValue`/`XxxVO`

* 领域事件：`XxxDomainEvent`

* 领域服务入参/出参：优先使用领域对象/值对象；禁止 `*Request/*Response/*DTO/*Command/*Query`

### Infrastructure（出站实现层）

* 持久化对象：`XxxEntity`（JPA）或 `XxxPO`（MyBatis），不复用 domain 模型

* 外部系统调用：`XxxClientRequest` / `XxxClientResponse`

* 端口实现：`XxxRepositoryImpl` / `XxxGatewayImpl`

## 结构化补充（映射与包）

* 映射器统一：`XxxMapper`（`toCommand/toDomain/toEntity/toResponse` 等）

* 建议包位：

  * `interfaces.web.dto` / `interfaces.facade.dto`

  * `application.command` / `application.query` / `application.dto`

  * `domain.model` / `domain.event`

  * `infrastructure.persistence.jpa.entity` / `infrastructure.client.dto`
